Instructions

Welcome to the FlightGear flight simulator user interface! If this is your first time, here are some very simple instructions to help you get started. Note that the instructions below are geared towards Mac OS users, though the installation and setup for Windows and Linux users should be similar.

Getting Started

  1. Download and install FlightGear. The instructions for doing that can be found here.
  2. Download the xml file here. This will be used for logging your flight dynamics simulation data.
  3. Navigate to Applications. This can be found by opening Finder and looking on the left panel. Alternatively, this can be done by opening a Terminal window and entering the following command: 
    > cd ~/../../Applications
  4. Now find FlightGear.app. Instead of double clicking on it to get it started, right click and select Show Package Contents.
  5. Now navigate to Contents > Resources > data > Protocol. Once again, this can alternatively be done by entering the following command in Terminal: 
    > cd ~/../../Applications/FlightGear.app/Contents/Resources/data/Protocol
  6. Copy and paste the xml file to this directory.

How to Use the Flight Simulator User Interface

  1. Find your computer's public IP Address. This can be done by a simple Google search.
  2. Go back to the FlightGear.app directory mentioned in the Getting Started section. Now navigate to Contents > MacOS. This should be done in Terminal by entering the following command: 
    > cd ~/../../Applications/FlightGear.app/Contents/MacOS
  3. Now enter the following command: 
    > ./fgfs --generic=socket,out,5,my_public_ip_address,9999,udp,userinterface
    where you will enter your public IP address into the my_public_ip_address space. This should start up FlightGear.
  4. There should be a file labeled udppacket.py in the same directory as your user interface. Open up this file in any sort of text editing software, and make the following edit to the fifth line: 
    UDP_IP = #my_public_ip_address
    by replacing #my_public_ip_address with your public IP Address.
  5. Open Terminal and navigate to the directory containing updpacket.py, and enter the command 
    > python udppacket.py
    This script will be responsible for logging the flight dynamics data.
  6. Congratulations! You are done! You can now use the user interface to retrieve your logging data or to view your live map. In the case that your live map does not appear and you are using Google Chrome, open a Terminal window and navigate to the executable version of Chrome: 
    > cd ~/../../Applications/'Google Chrome.app'/Contents/MacOS
    Now allow local file access by making the following command: 
    > ./'Google Chrome' --allow-file-access-from-files
